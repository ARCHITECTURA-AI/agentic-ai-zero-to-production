# Contributing to Agentic AI: From Zero to Production

Thank you for contributing to this cohort program! Below are instructions on how to set up your branch, run local checks, and submit code issues.

---

## 1. Branching & Submission Workflow
1. **Fork the Repository**: Always work in a personal fork.
2. **Branch Naming**:
   - For exercises: `exercise/module-<num>-<username>` (e.g. `exercise/module-02-richard`)
   - For capstone: `capstone/<option>-<username>` (e.g. `capstone/option-a-richard`)
3. **Drafting Pull Requests**: When submitting your capstone, mark the Pull Request as a **Draft** until your test suite is 100% green.

---

## 2. Coding Standards
* **Python version**: Python 3.10 is the target baseline runtime.
* **Typing**: Use standard type hinting for all custom classes, router functions, and state payloads.
* **Testing**: All exercises must have matching mock-backed tests in their respective `tests/` subfolder. Direct LLM calls must be stubbed or mocked.
* **Linting**: Run `ruff check .` to check for formatting or layout style guidelines before committing.

---

## 3. Getting Help & Submitting Feedback
* For course clarifications, starter bug reports, or capstone feedback, please create a GitHub issue using the appropriate template in this repository.
